== Accessing the Terminal

This workshop provides a standalone *Terminal* for running `oc` commands against the cluster. You can access it from the tabs on the right-hand side of this workshop guide.

. Select the *Terminal* tab at the top of the right-hand panel.
+
[.bordershadow]
image::02/terminal-access.png[The standalone Terminal tab in the workshop guide]

. Log in to the {ocp} cluster with your provided credentials:
+
[.console-input]
[source,bash,role="copypaste",subs=attributes+]
----
{login_command}
----
+
When prompted, answer `y` to accept the insecure connection:
+
[.console-output]
[source,text]
----

Welcome! See 'oc help' to get started.

== Import the PDF Processing Pipeline

Before you can trigger a pipeline run, you need to import the PDF processing pipeline into your Data Science Project. This is a one-time setup step.

=== Download the Pipeline File from Your Workbench

. In the {rhoai} Dashboard, navigate to your project and launch your *Workbench* (named "My Workbench"), or switch to it if it is already open.
+
[.bordershadow]
image::03/launch-workbench.png[Launching the Jupyter Workbench from the OpenShift AI dashboard]

. Once JupyterLab opens, use the file browser on the left to navigate to:
+
[.console-input]
[source,text]
----
hello-chris-rag-pipeline/lab-content/2.1/
----

image::02/pipeline-download.png[The main dashboard of Red Hat OpenShift AI after logging in]

. Inside this directory, you will find the file `simple-pdf-processing-pipeline.yaml`. *Right-click* on the file and select *Download* from the context menu. This will save the file to your local computer.

. You can now close the JupyterLab browser tab.

=== Upload the Pipeline to OpenShift AI

. In the {rhoai} Dashboard, navigate to the *Pipelines* section within your Data Science Project.

. Click the *Import pipeline* button.
+
image::02/pipeline-import.png[The Import pipeline button in the Pipelines section]

. Give your pipeline this exact name:
+
[.console-input]
[source,text]
----
simple-pdf-processing-pipeline
----
+
[IMPORTANT]
====
The name must match exactly. The event-driven trigger service looks up the pipeline by this display name.
====
+
image::02/pipeline-name.png[The pipeline name field filled in with simple-pdf-processing-pipeline]

. Click the *Upload* box and select the `simple-pdf-processing-pipeline.yaml` file that you just downloaded.
+
image::02/pipeline-upload.png[The pipeline upload dialog with the file selected]

. Click *Import pipeline* to finish.

Once imported, you are ready to trigger the event-driven flow in the next section.
